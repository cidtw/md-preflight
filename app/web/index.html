Promotion Preflight

등록 전 10초, 운영 리스크를 먼저 잡습니다

가격·기간·마진·재고·증정 조건을 결정론 규칙으로 대조합니다. ERP마다 컬럼명이 달라도 별칭으로 수렴해 같은 10개 룰을 적용합니다.

  • 10개 결정론 룰
  • 한글·동의어 헤더 지원
  • LLM은 서술만
  • 비로그인 검수 가능

워크플로

  1. 1파일 N개 업로드 · 역할을 프레임에 매핑
  2. 2헤더 별칭 정규화 + 룰 10개 판정
  3. 3이슈 · 체크리스트 · 매핑 리포트

입력 어댑터

파일 여러 개를 올린 뒤 역할 → 정규 프레임에 매핑합니다. 판정은 항상 행사·상품·재고 세 프레임으로 정제됩니다.

컬럼명·파일 역할이 달라도 됩니다

헤더는 별칭으로 수렴하고, 파일은 자동 추정 후 수동 확정합니다. 설정에서 별칭·임계값을 확인하세요.

설정 · 별칭 한글 샘플

아직 업로드된 파일이 없습니다. dirty 샘플 3종을 한 번에 올려 역할을 확인해 보세요.

켜짐 예) "행 12 · 할인율 45%가 마진 하한선을 초과합니다 → 할인율을 30% 이하로 조정하세요."

꺼짐 예) "MARGIN_FLOOR · error · 상품 A1023" 표준 문구만 · 오프라인 가능

샘플로 바로 체험

영문 dirty 한글 alias 한글 마스터 한글 재고 clean (이슈 0)

    교육 가이드

    1. 프로모션 · 마스터 · 재고 3파일을 올립니다.
    2. 컬럼명이 달라도 별칭 표의 정규 키로 수렴됩니다.
    3. 검수 결과의 이슈·체크리스트를 보고 원본을 수정합니다.
    4. (선택) 로그인하면 이력 대시보드를 볼 수 있습니다.
