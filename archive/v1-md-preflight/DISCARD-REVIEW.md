# 백엔드·경로 리뷰 — Keep / Discard / Defer

작성: 2026-07-14 · 근거: `2026-07-14-Project-Redesign.md` §1–3

## 판정 기준

| 판정 | 의미 |
|------|------|
| **KEEP** | 활성 모듈 파이프라인에 개념·코드로 남김 |
| **DISCARD** | 활성 트리에서 제거 (복원은 git 태그) |
| **DEFER** | 재설계 세부 단계에서 재도입 여부 결정 |

## 결과 표

| 항목 | 판정 | 이유 |
|------|------|------|
| 3단 파이프라인 개념 (입력→분석→출력) | **KEEP** | 재설계 핵심 골격 |
| FastAPI 앱 셸 · CORS · health | **KEEP** | 모든 환경 동일 경험 진입점 |
| Pydantic 스키마·설정 패턴 | **KEEP** | 간결·명시적 계약 |
| 결정론 판정 원칙 (LLM이 점수 조작 금지) | **KEEP (원칙)** | 새 가중치 엔진에도 적용 |
| 10룰 프로모션 엔진 (`app/rules`) | **DISCARD** | v1 도메인; 스코프 과다 |
| Excel/CSV ingest·별칭·멀티시트 | **DISCARD** | “파라미터 입력” 방향과 불일치 |
| 역할 매핑 / detect-roles | **DISCARD** | 입력 복잡도 증가 |
| LLM 서술 이중 어댑터 | **DISCARD** | 1차 스켈레톤 불필요; 한 줄 추천은 템플릿으로 |
| Clerk / Neon / History / RunStore | **DISCARD** | 가벼운 서비스 대비 과부하 |
| `app/sources` 스텁 | **DISCARD** | 실체 없는 표면 |
| SPA 전체 (편집기·이력·테마·설정 카탈로그) | **DISCARD** | 웹 무거움 피드백 직격 |
| 샘플 CSV/XLSX 세트 | **DISCARD (활성)** | 태그·데이터 히스토리에 존재 |
| 평가 가중치 레지스트리 | **DEFER** | 조사 완료 후 채움 (스켈레톤 자리만) |
| 실 서비스 도메인 파라미터 정의 | **DEFER** | 판 제작 이후 |
| UI 프레임워크 선택 | **DEFER** | 모듈 백엔드 안정 후 |

## 모듈형 백엔드 목표 구조 (사전 작업 결과)

```
app/
  pipeline/
    input/      # 파라미터 템플릿 수신·검증
    analyze/    # 사전 구성 가중치 기반 점수
    output/     # 한 줄 recommendation + 구조화 결과
  api/          # 얇은 HTTP 어댑터
  core/         # 설정·에러
  schemas/      # 요청/응답 계약
  web/          # 최소 플레이스홀더 (세부 UI 이후)
```

오케스트레이션은 `pipeline.runner.run()` 한 경로만 호출한다.  
향후 기능은 **스테이지 내부 플러그인** 또는 **신규 스테이지 모듈**로 추가한다.
