# 방향성 — 재설계

## v1에서 무엇이 문제였는가

1. **너무 많은 기능** — 검수 엔진·어댑터·인증·이력·대형 SPA가 한 제품에 공존  
2. **서비스 근거의 빈약** — 조사 기반 평가 가중치·추천 논리가 제품 중심에 없음  
3. **가벼운 가치 vs 무거운 셸** — 웹표준·프로덕션 배선을 지킨 결과, 본체가 얇은데도 체감 부하 증가  

## 재설정 목표

| 축 | 목표 |
|----|------|
| 스코프 | 좁고 짜임새 있게 |
| 입력 | 파일 3종 강제 업로드가 아니라 **파라미터 템플릿** |
| 분석 | 사전 조사를 거친 **평가 가중치** (결정론) |
| 출력 | **명확한 한 줄 recommendation** (+ 필요 시 구조화 근거) |
| 구조 | 블록·모듈형 3단 파이프라인 |
| 경험 | 어느 환경에서든 쾌적·동일 |

## 비목표 (지금 하지 않음)

- v1 프로모션 10룰·Excel 조인 복원  
- Clerk/Neon/LLM 멀티 프로바이더 재배선  
- 풀 SPA·CSV 인페이지 편집기
